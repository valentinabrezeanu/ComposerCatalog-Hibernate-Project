# ComposerCatalog-Hibernate-Project
aplicație web modernă concepută pentru gestiunea bazelor de date, a cărei arhitectură se sprijină pe framework-ul Hibernate. Utilizarea acestei tehnologii de tip ORM a permis transformarea tabelelor din MySQL în obiecte Java, eliminând interogările SQL manuale și facilitând administrarea automată a relațiilor complexe de tip many-to-many.
Tehnologii utilizate

Pentru proiectarea și administrarea bazei de date a fost utilizat MySQL Workbench 8.0, o soluție consacrată în domeniul dezvoltării software, recunoscută pentru fiabilitatea sa în gestionarea bazelor de date relaționale. Alegerea acestei tehnologii a fost determinată de necesitatea modelării corecte a relațiilor de tip many-to-many dintre entitățile „Compositions” și „Composers”, garantând integritatea și coerența datelor prin intermediul unui tabel de legătură dedicat colaborărilor.
La nivelul interfeței a doua tehnologie folosita  a fost Hibernate ORM, reprezentând un cadru (framework) de tip open-source dedicat limbajului Java, specializat în maparea obiect-relațională (ORM). Rolul său principal este de a facilita interacțiunea dintre modelul de date orientat pe obiecte al aplicației și sistemul de gestiune a bazelor de date relaționale, rezolvând astfel problema „nepotrivirii de impedanță”. Prin utilizarea Hibernate, accesul direct la baza de date prin interogări SQL complexe este înlocuit cu funcții de manipulare a obiectelor la nivel înalt, ceea ce sporește eficiența dezvoltării și mentenanța codului.
Din punct de vedere arhitectural, Hibernate acționează ca un strat intermediar (middleware) care face legătura între logica de business a aplicației și motorul bazei de date. Acesta utilizează API-uri standard din ecosistemul Java, precum JDBC (Java Database Connectivity), JTA (Java Transaction API) și JNDI, pentru a gestiona persistența datelor. Astfel, tehnologia permite executarea automată a operațiilor de tip CRUD (Create, Read, Update, Delete), asigurând o gestiune performantă a stării obiectelor și a tranzacțiilor în cadrul sistemului.
Implementare

Baza de date a fost realizată cu ajutorul MySQL Workbench, fiind alcătuită din două tabele principale: Compositions și Composers. Pentru a realiza o asociere de tip M:N (Many-to-Many) între aceste două entități — având în vedere că o compoziție poate fi rezultatul muncii mai multor autori, iar un compozitor poate contribui la multiple opere — a fost creată o tabelă intermediară numită Collaborations.
